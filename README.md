# Стационар (1С:Учёт пребывания пациентов)

Конфигурация 1С:Предприятие 8.3 для автоматизации стационарного и дневного отделения медицинского учреждения: учёт госпитализаций, назначение лекарств и питания, списание расходов, расчёт стоимости пребывания и финансовая отчётность.

Исходники выгружены в XML-формате штатным механизмом платформы и версионируются через git.

## О проекте

Репозиторий содержит исходники конфигурации, выгруженные через **Конфигурация → Выгрузить конфигурацию в файлы...**. Формат позволяет вести историю изменений в git, делать code review по XML-диффам и работает на базовых версиях платформы без хранилища конфигурации.

| Параметр | Значение |
|---|---|
| Внутреннее имя | `Конфигурация` |
| Режим запуска по умолчанию | Управляемое приложение |
| Совместимость расширений | Версия 8.3.27 |
| Язык по умолчанию | Русский |
| Вариант встроенного языка | Russian |

## Состав конфигурации

### Справочники

| Справочник | Назначение | Реквизиты |
|---|---|---|
| `Пациенты` | Картотека пациентов | `ФИО`, `ДатаРождения`, `Пол`, `КонтактныеДанные`, `Контакты`, `ПолисОМС`, `Льготы` |
| `Палаты` | Палаты стационара | `Номер`, `Отделение`, `КоличествоКоев` |
| `ОтделениеСтационара` | Отделения больницы | `Местоположение`, `Ответственный` |
| `Сотрудники` | Персонал учреждения | `ФИО`, `Должность`, `ДатаРождения` |
| `МедицинскиеУслуги` | Перечень медицинских услуг | `Цена`, `Категория` |
| `ЛекарственныеПрепараты` | Справочник медикаментов | `ЕдиницаИзмерения`, `Производитель`, `Цена` |
| `Питание` | Рационы и блюда | `МенюБлюд`, `ВремяПриема`, `Цена` |
| `Пол` | Классификатор пола пациента | `НазваниеПола` |

### Документы

**ПребываниеПациента** — основной документ госпитализации: фиксирует факт помещения пациента в палату и назначения на период пребывания.
- Шапка: `Пациент`, `Палата`, `Дата1`, `Ответственный`
- Табличная часть «Лекарства»: `Лекарство`, `Доза`, `Количество`
- Табличная часть «Услуги»: `Услуга`, `Количество`
- Табличная часть «Питание»: `Питание`, `Количество`

**ПриемВДневнойСтационар** — приём пациента в дневной стационар (без ночного пребывания).
- Шапка: `Пациент`, `ДатаНачала`, `ПланированиеДатаОкончание`, `Врач`, `Палата`

**СписаниеМедикаментовИПитания** — документ фактического расхода лекарств и питания по пациенту.
- Шапка: `Пациент`, `Дата1`
- Табличная часть «СписанныеЛекарства»: `Лекарства`, `Количество`
- Табличная часть «СписанноеПитание»: `Питание`, `Количество`
- При проведении формирует движения по регистрам `РасходМедикаментов` и `РасходПитания`

**РасчетСтоимостиПребывания** — итоговая калькуляция затрат за период.
- Шапка: `Пациент`, `Период`, `СтоимостьУслуг`, `СтоимостьЛекарств`, `СтоимостьПитания`, `ОбщаяСтоимость`

**ВыпискаПациента** — документ закрытия госпитализации.
- Шапка: `Пациент`, `ДатаВыписки`, `ИтоговаяСтоимость`, `Заключение`

### Регистры накопления

| Регистр | Тип | Измерения | Ресурсы |
|---|---|---|---|
| `РасходМедикаментов` | Остатки | `Пациент`, `Лекарства`, `Дата` | `Количество` |
| `РасходПитания` | Остатки | `Пациент`, `Питание`, `Дата` | `Количество` |

### Отчёты

| Отчёт | Назначение |
|---|---|
| `ФинансовыйОтчет` | Финансовые показатели по периоду |
| `ОтчетПоЗагрузкеИДоходам` | Загрузка палат и доходы учреждения |

## Бизнес-процессы

**Госпитализация (круглосуточный стационар):**
```
ПребываниеПациента (назначения)
       │
       ├─→ СписаниеМедикаментовИПитания (фактический расход)
       │         ├─→ AccumulationRegister.РасходМедикаментов
       │         └─→ AccumulationRegister.РасходПитания
       │
       ├─→ РасчетСтоимостиПребывания (калькуляция затрат)
       │
       └─→ ВыпискаПациента (закрытие госпитализации)
```

**Дневной стационар:**
```
ПриемВДневнойСтационар
       │
       └─→ РасчетСтоимостиПребывания → ВыпискаПациента
```

**Финансовый контроль:**
```
AccumulationRegister.РасходМедикаментов ──┐
AccumulationRegister.РасходПитания ───────┴─→ ФинансовыйОтчет
                                              ОтчетПоЗагрузкеИДоходам
```

## Требования

- Платформа 1С:Предприятие 8.3 (версия 8.3.27 и выше)

## Обновление репозитория после изменений конфигурации

1. Откройте базу в режиме **Конфигуратор**.
2. Выполните **Конфигурация → Выгрузить конфигурацию в файлы...**, указав папку репозитория — изменятся только реально изменённые файлы.
3. Зафиксируйте изменения в git:

```bash
git add .
git commit -m "Описание изменений"
git push
```

## Развёртывание конфигурации из репозитория

1. Создайте новую информационную базу в 1С (или используйте существующую).
2. В Конфигураторе выполните **Конфигурация → Загрузить конфигурацию из файлов...**, указав путь к папке репозитория.
3. Обновите конфигурацию базы данных (**F7**).
